# 常见问题

## 1. 没有域名，能不能直接用

可以。

当前推荐默认就是：

- `HTTP + nginx 反代`
- 通过服务器 IP 直接访问

这适合先把系统跑起来，但不会有浏览器原生信任的 HTTPS 证书。

## 2. 没有域名就不能部署吗

不是。

没有域名仍然可以部署和使用，只是：

- 你访问 WebUI 时是 HTTP
- 如果未来要浏览器无警告 HTTPS，还是需要域名

## 3. 我需要手动编辑 `.env` 吗

通常不需要。

对于全新 Ubuntu 服务器，bootstrap 会自动生成一份可用的 `.env`，并设置随机管理员密码。只有在你想覆盖默认值时，才需要在部署前传环境变量。

## 4. 这个项目支持多条路由同时运行吗

支持。

这正是当前版本的重要设计点之一。不同路由可以拥有不同的：

- B 落地
- C 监听口
- 当前选中的 A 节点

并且可以并行运行。

## 5. 导出的 `socks5://` 地址为什么不是我想要的公网 IP

通常是因为自动探测到的地址不符合你的实际公网入口。

可以通过：

- 部署时传 `CSG_PUBLIC_HOST`
- 或在全局设置里调整导出主机地址

来修正默认导出结果。

## 6. 要放行哪些端口

最少通常需要：

- `18080/tcp`
- `80/tcp`
- 你实际使用的 `Socks5 C` 端口，例如 `10808/tcp`

如果云防火墙没放行，服务器内服务启动成功也不等于外网一定可达。

## 7. 对公网开放的 Socks5 现在默认有认证吗

是的。

当前逻辑会尽量避免把公网 Socks5 做成匿名开放代理。如果你把监听地址放到公网，应当始终保留用户名密码。

## 8. 为什么我能打开 WebUI，但最终导出的 C 不能用

这是很常见的情况，因为 WebUI 可访问只代表面板服务正常，不代表整条代理链路已经跑通。

建议按顺序检查：

1. A 订阅是否可用
2. 当前选中的 A 节点是否可用
3. B 落地是否可用
4. C 端口是否已经对外放行

## 9. `mihomo` 没有自动下载成功怎么办

常见原因是：

- 服务器访问 GitHub Release 失败
- 网络环境限制
- 临时下载异常

这不一定会阻止整个 WebUI 启动。你可以稍后在 UI 内再次执行安装，或者手动处理网络问题后重试。

## 10. 如何查看服务日志

```bash
sudo journalctl -u clash-socks-webui -n 200 --no-pager
sudo systemctl status clash-socks-webui --no-pager -l
```

如果你还想看监听端口：

```bash
sudo ss -lntp
```

## 11. 能不能部署到非 Ubuntu 系统

当前不建议把现成脚本直接用于非 Ubuntu 系统。

项目默认安装流程、依赖安装和服务管理都围绕 Ubuntu 编写。如果你想适配其他发行版，最好先自行修改安装器。

## 12. 这个项目会自动帮我配置 HTTPS 吗

默认不会。

因为没有域名时也应该能先跑起来，所以默认只做了 `nginx` 反代。真正的浏览器受信任 HTTPS 仍然需要：

- 你有域名
- 你为 `nginx` 或 `caddy` 配了证书
