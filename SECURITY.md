# Security

这份文档说明项目当前默认做了哪些安全加固，以及部署者仍然需要自己负责哪些部分。

## 当前默认安全策略

当前版本默认会做这些事情：

- 关闭 FastAPI 的 `/docs` 和 `/openapi.json`
- 让 WebUI 默认运行在 `nginx` 后面，而不是直接把 `uvicorn` 暴露到公网
- 让 `uvicorn` 默认只监听本机 `127.0.0.1`
- 对公网 Socks5 监听强制要求认证
- 收紧 `.env` 和运行时文件的权限
- 通过 `systemd` 托管 WebUI
- 启动时清理残留 `mihomo` 进程并恢复已跟踪路由

这些默认值的目标不是“自动满足所有安全要求”，而是尽量避免最常见、最容易被忽略的暴露面。

## 你仍然需要自己负责的部分

### 1. 云防火墙 / 安全组

项目不能替你修改云平台控制台里的安全组规则。

请只开放你真正需要的端口，例如：

- `18080/tcp`
- `80/tcp`
- 业务需要的 `Socks5 C` 端口

### 2. 管理员凭据

部署脚本会随机生成管理员密码，但你仍然需要：

- 妥善保存它
- 避免在公共记录里泄露
- 在必要时主动轮换

如果你选择手动指定密码，不要使用弱口令。

### 3. 域名与 HTTPS

没有域名时，默认部署模式是：

- `HTTP + nginx 反代`

这足够先把项目跑起来，但不是浏览器受信任的 HTTPS。

如果你要更正式的公网管理入口，请：

- 绑定域名
- 为 `nginx` 或 `caddy` 配证书

### 4. 最终业务端口暴露

如果某个 `Socks5 C` 端口不需要公网开放，就不要开放。

如果需要公网开放，请确认：

- 口令确实已设置
- 监听地址和端口符合你的预期
- 云侧只放行必要来源

## 为什么要强制公网 Socks5 认证

一个不需要认证的公网 Socks5 端口，很容易演变成匿名公开代理入口。

这既会消耗你的服务器资源，也会把出口行为暴露在你的 IP 上。因此，当前逻辑对公网监听要求账号密码成对存在。

## 为什么 docs 默认关闭

这个项目更像运维面板，而不是开放给第三方的开发者平台。

在公网环境里保留：

- `/docs`
- `/openapi.json`

通常只会增加额外暴露面，而不会给最终使用者带来明显收益。

## 运维建议

- 定期升级服务器系统包
- 定期更新项目版本
- 检查 `mihomo` 是否能正常更新
- 为关键数据做备份
- 避免在聊天记录、工单系统或公共页面里直接贴真实口令
- 在多人协作场景中避免共享同一套长期口令

## 已知边界

当前项目不提供这些企业级安全能力：

- 多角色权限控制
- SSO / OAuth
- 审计日志中心化
- 自动证书生命周期管理
- WAF / DDoS 等外围安全能力

如果你需要这些能力，应当把它放在更完整的外围基础设施后面使用，而不是把本项目当成完整安全平台。

## 发现安全问题时怎么做

如果你发现的是：

- 文档问题
- 配置误导
- 一般性 bug

可以通过普通 issue 或 PR 反馈。

如果你发现的是：

- 可直接利用的高危漏洞
- 真实可复现的公开代理或凭据泄露风险

建议不要先在公开 issue 里贴完整利用细节，而是先联系维护者或先给出最小必要信息，避免在修复前扩大风险。
